# 📋 VendorMap 設定範例展示

## 🎯 系統解析規則

### ✅ 支援的格式
- **基本格式**：`品項名稱 數量`
- **含屬性格式**：`品項名稱 屬性1 屬性2 數量`（數量必須在最後）

### ❌ 不支援的格式
- 單位在數字後面：`雞蛋 2盒` ❌（最後必須是數字）
- 只有品項名稱：`雞蛋` ❌（必須有數量）

## ⚠️ 常見錯誤

### ❌ 錯誤範例 1：單位在數字後
```
全聯
  雞蛋 2盒    ← 錯誤！最後必須是數字
  牛奶 1瓶    ← 錯誤！
```

### ✅ 正確寫法
```
全聯
  雞蛋盒 2    ← 正確：單位在名稱中
  牛奶瓶 1    ← 正確
```

或更簡單：
```
全聯
  雞蛋 2     ← 正確：不寫單位
  牛奶 1     ← 正確
```

### ❌ 錯誤範例 2：缺少數量
```
全聯
  雞蛋       ← 錯誤！必須有數量
  牛奶       ← 錯誤！
```

### ✅ 正確寫法
```
全聯
  雞蛋 0     ← 正確：數量可以是 0
  牛奶 0     ← 正確
```

或使用 `-` 開頭（數量預設為 0）：
```
全聯
  - 雞蛋     ← 正確：預設數量 0
  - 牛奶     ← 正確
```

## 💡 屬性組合的彈性說明

### ✅ 系統完全支援的屬性組合方式：

1. **單一屬性**：`T恤 黑 10`
2. **多個屬性**：`T恤 黑 M 10`
3. **複雜屬性**：`珍珠奶茶 大杯 正常甜 正常冰 10`
4. **加料選項**：`雞腿飯 加滷蛋 加青菜 10`
5. **品牌規格**：`維他命C 品牌A 1000mg 10`
6. **任意組合**：可以自由組合任意數量的屬性

### 🎯 關鍵原則：

- **數量必須在最後**：所有屬性都放在品項名稱中，數量放在最後
- **空格分隔**：屬性之間用空格分隔
- **完整保留**：系統會完整保留所有屬性資訊，不會遺失
- **精確匹配**：使用者下單時，品項名稱必須完全匹配（包含所有屬性）

### 📝 實際使用範例：

**Owner 設定**：
```
飲料店
  珍珠奶茶 大杯 正常甜 正常冰 10
  珍珠奶茶 大杯 半糖 少冰 5
```

```

  珍珠奶茶 大杯 半糖 少冰 1 (小華)
```

## 📝 總結

1. **格式規則**：`廠商/店家\n  品項名稱 數量`
2. **品項名稱**：可以包含屬性（顏色、尺寸等），但數量必須在最後
3. **數量**：必須是數字，放在品項名稱的最後
4. **單位**：如果要使用單位，放在品項名稱中（如「雞蛋盒」），不要放在數字後
5. **彈性**：系統支援多種格式，可以根據需求自由設定
